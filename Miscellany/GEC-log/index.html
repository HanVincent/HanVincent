Paper

Hybrid Systems and Type Filtering

  • Error Type

    • Open-class: spelling, collocation
    • Closed class: determiners, preposition

  • Hybrid System

    • Ruled-based system(RBS)
      • Self-Assessment and Tutoring(SAT) system
        • text assessment
        • sentence evaluation
        • word-level feedback
      • Only use word-level feedback of SAT (∵ on strict correction not detection)
        • Rules from n-gram error > 5 times & 90% in CLC
        • Rules from machine-readable dict.
    • Ad-hoc statistical machine system(SMT)
      • Try to ‘translate’ incorrect into correct
      • Many parallel corpora
      • Lexical reoredering model
        • Pialign 2011 (word alignment)
        • charactr-level Levenshtein distance
      • 4-gram target language model by Kneser-Ney smoothing
        • Knser-Ney smoothing: 不僅僅看 unigram，該字的出現機率正比於前一字的單字種類數
      • Decoding
        • Moses 2007 to generate 10 correctionss
        • Moses: statistical machine translation with beam search and phrase-based translation …
      • NLTK for NLP

  • Ranking

    • Combining all possible corrections and ranking
    • Ranking for alternative corrections
      • Target language model in SMT
      • M$ web n-gram services based on 5-gram
    • M$ is better evaluated by $M^2$ Scorer(2012) due to massive data

  • pre-Analysis

    • (normalisation: bias towards prefer deletion or skip insertion)
    • Performance
      • Based on Error Type
      • Lower down due to unnecessary correction -> recall ↓(?) -> remove types that precision = 0
      • Error type filtering technique
        1. Extract correct and original sentence
        2. Delete unwanted types
        3. Edit original text from reference above

  • Experiment

    • Order: RBS > SMT
    • Best: RBSc > LM > SMT_10-best > LM
    • Good at
      • mechanical errors(punct, capitalization, and spelling)
      • enforcing diff. types of agreement
    • Bad at
      • others, reordering, srun (precision = 0)
      • Wci(idioms, collocation)
      • …

  • Result

    • Score for unnecessary deletion (could)
    • Domain diff. (social media > the media)
    • Uncredited correction (right but diff. from answer, right in isolation but wrong)
