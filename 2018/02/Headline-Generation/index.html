Headline Generation Using Recurrent Neural Networks

Posted on 2018-02-13 | In Paper Notes | | Views:

Paper Link

Abstract

  • 使用 encoder-decoder LSTM model with attention mechanism
  • 用 AWS, TensorFlow, English Giga-word 作為工具

Introduction

  • summarization 的兩種方法：extractive 和 abstractive
    • extractive: 擷取出 main point
    • abstractive: 換句話說
  • 不同於 text summarization 的格式，一般新聞標題通常較令人吃驚，但是 Gigaword 比較偏向新聞內容的摘要

Related Work

  • 這篇為上篇Generating Headlines的實作及改良
  • 用attention=based, encoder, latent soft 產生 abstractive summaries 的 data-driven 方法
  • instrinsic: 直接找使用者判斷
  • extrinsic: 注重效能大於分析，task-based

Dataset

  • English Gigaword 2nd edition
  • 同個標題有很多不同的文章
  • 重複很多相同的新聞篇幅
  • 因為以上問題，這篇論文使用 1994-2004 APW 的新聞
  • 資料以 Standard Generalized Markup Language 格式儲存
  • 每筆資料包含 meta-data(獨立ID和文件類型)、標題、文章

Preprocessing

  • 使用 python BeautifulSoup4 進行解析
  • 取出標題及第一段段落
  • 以下資料刪除以提升正確率、運算效率及節省記憶體空間：
    • 標題不符合文法或包含非英文字元
    • 標題含有’-‘或者第一段少於三個字
    • 標題有不完整的括號
    • 含有不完整的 SGML-tagged 或遺失重要 tag

Word Length Bins

  • 目標是藉由優化 bin sizes 達到縮小標題和句子間的 padding
  • 將標題長度以: 10, 12, 15, 18 做級距切割，第一句長度以: 20, 30, 40, 45, 50, 60, 70 做級距切割，觀察不同間距的頻率分佈

Encoder-decoder LSTM Model

  • 使用 encoder-decoer LSTM model (RNN) 和 attention mechanism
  • 使用兩層（上篇用一層）
  • 使用五個 buckets 和被優化的 bucketing 長度來減少被用在句子裡的 paddings 數量（上篇只用一個 bucket）

  • 先顛倒文章第一句句子順序後當成 input 放入 encoder，如此 input 的前幾個字能比較靠近 output 的前幾個字，以抓出 short term dependencies

    …其實如果 encoder 是 RNN 的話，理論上越後輸入的單詞影響越大，並非等權的，估計這也是為何 Google 提出 Sequence to Sequence 模型時，發現把輸入句子逆序輸入做翻譯效果會更好的原因）

  • 使用 greedy decoder 取代 beam search decoder 來產生標題

  • 被產生出來的字會被當成下一個預測字的 input
  • attention mechanism 給予 encoder 裡的 ouput hidden states 和 current hidden state 權重，來決定哪個字比較重要
  • dropout = 0.2, hidden units = 512, word embeddings = 512 dim
  • 高頻 80,000 words，其他字用 _UNK 或 OOV(out-of-vocabulary)
  • 使用五個 buckets ((30, 10), (30, 20), (40, 10), (40, 20), (50, 20)) (sen_len, head_len)，意指有五個不同的 encoder-decoder LSTM models
  • 使用 _PAD 來填充句子和標題到達 bucket size

Result

  • 70% 訓練集，20% 評估集，10% 測試集
  • batched 訓練大小 = 128
  • learning rate = 0.5
  • gradient descent optimizer
    • decay factor = 0.99
    • maximum gradient norm = 5
  • 10 epochs
  • 能使用沒在句中出現過的字來產生好的標題
  • 遺漏訊息、產生錯誤訊息、因為 bucketing issue 導致出現 前就結束，最後一個問題可被解決，只要eos還沒出現，就不斷轉成更大的 bucket

Evaluation

  • 使用 system-oriented 和 user-oriented 的評估方式來比較結果
  • user evaluation
    • one-factor within-subject design evaluation survey
    • 變因為標題產生方式的不同
    • 使用 Google 表單進行調查
      • 搜集性別、年齡、國籍、母語，以確認國籍和母語不影響受試者對句子的評估
      • 提供10篇新聞讓使用者閱讀，並提供真實的和預測的兩種標題
      • 以1-5分各自評分兩者標題
      • 再以1-5分評分兩者間的相似度
      • 最後再要求受試者選出有描繪到主題的標題
  • system-oriented automatic evaluation
    • 使用 BLEU 評斷兩者間的相似程度
    • BLEU 是以判斷 n-grams 重疊程度的 precision-based 衡量方法
    • 使用 k = 1 (unigram)
  • 為了比較系統評估和人工評估之間的關係，挑 BLUE 分數在 0.1 - 0.8 之間的新聞作為人工評估的資料
  • 以 paired samples t-test 來比較使用者對於兩者標題總結的優劣

  • 相似度方面，使用 Spearman 相關來衡量兩者評估方式間的關聯

    Pearson 相關常用來呈現連續型(continous)變數之間的關聯性，尤其在變數符合常態分配的假設下，最爲精確；而spearman相關則不需符合常態，僅要求變數的資料型態至少爲有序的(ordinal)。

Data Analysis

  • 樣本分析
    • 71人參加，女性42人(59%)，男性29人(41%)
    • 平均年齡 28.52歲，標準差 9.020
    • 國籍：美國 34人(47.8%)，台灣 29人(40.8%)，墨西哥 2人，中國 2人，印度 2人，加拿大 1人，馬來西亞 1人
    • 母語：英文 35人，中文 33人，印度語 2人，西班牙 1人
  • 兩者標題的評估比較
    • 使用 independent sample t-test 去分析是否母語會影響兩者標題總結優劣的評估結果，結果呈現無太大影響
    • 從 paired sample t-test 比較，有 7 則新聞預測的標題比較好
  • 系統和人工的評估比較
    • 使用 Spearman’s rank-order correlation
    • 系統和人工評估之間的關聯呈現微弱且正向，表示兩者並無一致性的結果
  • 優良標題的選擇
    • 結果顯示，10則新聞預測標題和實際標題各佔半
